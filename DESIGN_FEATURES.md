# 设计特性文档 (Design Features)

本文档总结了 Mermaid 可视化编辑器的核心设计决策、疑难杂症解决方案和技术特性。

## 1. 架构设计：完全解耦的双向同步

### 为什么选择解耦？
早期的架构中，代码区和画布区采用自动监听同步（effect 依赖），但这会导致以下问题：
- **循环依赖**：代码改变触发画布更新 -> 触发重绘 -> 触发设置同步回代码 -> 再次触发更新。
- **状态丢失**：用户正在编辑节点属性时，由于另一个组件的渲染导致整个画布重新解析，丢失焦点和坐标。
- **性能问题**：频繁调用 Mermaid.js 的原生 API 和 Dagre 布局非常消耗性能。

### 解决方案
1. **渲染区独立**：使用原生的 Mermaid.js 只读渲染，保证图表无论多复杂，渲染结果都是 100% 官方标准的。
2. **画布区独立**：使用自定义的 SVG 画布和 `graphEditorStore` 独立管理交互状态。
3. **手动同步**：
   - ⬇️ **读取代码**：清除当前所有临时编辑状态 -> 从源码重新进行正则解析 -> 使用 Dagre 进行空间坐标布局 -> 更新到 Zustand Store。
   - ⬆️ **回写代码**：遍历 Store 中的节点、边、方向、曲线样式 -> 拼接组装标准的 Mermaid 源码 -> 覆盖代码编辑器。

## 2. 边连接算法 (Edge Connection Algorithm)

### 挑战
在拖拽节点时，连接两条边的路径应该看起来自然，不会生硬地穿过节点内部。

### 智能连接点选择
1. 对于每一对连接的源节点和目标节点，分别取其上下左右 (Top, Bottom, Left, Right) 4 个中点。
2. 组合出 16 种连接可能性 (4x4)。
3. 通过两点间直线距离计算基础距离。
4. **方向权重惩罚机制**：
   - **相对位置权重**：如果两个节点是上下分布（纵向高度差 > 横向宽度差），优先鼓励垂直连接（top/bottom）；如果是左右分布，优先鼓励水平连接（left/right）。匹配时基础距离 `* 0.8` 打折。
   - **布局方向权重**：用户设置的 Mermaid 整体方向（TB/BT vs LR/RL）对线条走向有决定性影响。如果连线的终点连接面与图表全局布局方向一致，基础距离再次 `* 0.8` 打折。
5. 取折后距离最短的一组连接点。

### 平行边智能分离 (Parallel Edge Offset)
当用户在两个相同节点之间创建了多条方向相同的边时（例如双向流），如果坐标重叠会导致视觉上只显示一条。
- 解决：通过统计 `edges` 中 `source-target` 相同的集合，计算其 `parallelCount` 和 `parallelIndex`。
- 如果有多条边，则基于当前边的索引和基础间距（15px），沿节点边界面的垂向进行物理偏移（Offset）。

## 3. 高级图形支持

### 形状映射字典 (Shape Mapping)
内部解析器 `graphParser.ts` 实现了完整的正则匹配，将 Mermaid 语法（如 `A((Text))`，`A{Text}`）或者 v11 的扩展注解（`@{ shape: bolt }`）精准映射到内置的近 50 种标准图形。

### 曲线样式的支持
为了实现原生 Mermaid 中 `%%{init: {'flowchart': {'curve': 'step'}}}%%` 这种配置：
- 解析时会预先进行正则扫描，提取首部的 `curve` 字段。
- SVG 中使用了原生的 `d` 属性构造器，将 `curveStyle` 实时翻译为不同的贝塞尔控制点或正交折线 (`step`, `stepBefore`, `stepAfter`)。

## 4. 跨域安全的图像导出

### 污染的 Canvas 问题 (Tainted Canvas)
调用 `canvas.toBlob()` 时，由于原生的 Mermaid SVG 中通常会包含字体、`foreignObject` 等外部引用，浏览器的安全策略会将其标记为 tainted，从而拒绝导出图像。

### 解决方案
在 `MermaidPreview.tsx` 中实现：
1. `cloneNode(true)` 深拷贝 SVG 节点。
2. 扫描并 `remove()` 内部所有的 `foreignObject`。
3. 动态计算 SVG 的 BBox 尺寸，并设定 2 倍的高清缩放比。
4. 使用 `XMLSerializer` 将清理后的 DOM 转为字符串。
5. **核心**：使用 `data:image/svg+xml;charset=utf-8` 编码，而不是原生的 `Blob URL` 传入 Image.src，完美绕过跨域污染。
